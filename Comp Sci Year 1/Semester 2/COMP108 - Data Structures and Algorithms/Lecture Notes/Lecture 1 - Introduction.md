## Why Learn Data Structures and Algorithms?
An algorithm is a set of instructions that allows a person or a computer to complete a task. We need algorithms for all sorts of tasks:
- Deciding on a restaurant to go to requires some kind of **ranking** algorithm
- Determining if an ATM can disperse a given amount of money and which bank notes it must disperse requires a **combinatorial algorithm**
- Determining what the shortest path is between points A and B requires a **pathfinding** algorithm. In this case we need the shortest path so the efficiency of the algorithm matters.

There are many more examples, but that is enough to illustrate the point.

This module focuses on algorithm design and appropriate use of data structures. This is foundational for building efficient and effective programs. These sorts of things are incredibly important if we want to process data quickly, or want to process massive amount of data?

The module aims are:
- To introduce the notation, terminology, and techniques underpinning the study of algorithms
- To introduce basic data structures and associated algorithms
- To introduce standard algorithmic design paradigms and efficient use of data structures employed in the development of efficient algorithmic solutions
