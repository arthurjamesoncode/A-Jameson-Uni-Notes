## The Most Basic Datatypes
### The Natural Numbers
The natural numbers are all of the positive integers and 0. Whether 0 is a natural number or not depends on the context but for the purposes of the module it is.

The formal definition of natural numbers is this:
>A natural number is any number that can be obtained from 0 by applying the operation S(n) = n+1 some number of times.

>The notation for the set of naturals is $\mathbb{N}$.
### The Integers
The integers are the set of all the whole numbers. They are a superset of the natural numbers, and include negative numbers. I don't think a formal definition is necessary.

>The notation for the set of integers is $\mathbb{Z}$.
### The Rational Numbers
The rational numbers are any number that can be written as fraction.

The formal definition of the rational numbers is this:
>A rational number is any number that can be written as $\frac{m}{n}$ where m and n are both integers.

This is a superset of the integers since any integer n can be written as $\frac{n}{1}$.

>The notation for the set of rationals is $\mathbb{Q}$. 
### The Real Numbers
The real numbers are any number used in measuring a one-dimensional quantity. For example: length of a line, temperature, and speed.

This is a superset of the rational numbers, and contains irrational numbers as well such as $\sqrt{2}$ or $\pi$.

>The notation for the set of reals is $\mathbb{R}$
## Mathematical Proof
Maths underpins STEM subjects. In a lot of cases, we are concerned with solving and computing.

>However what is also important is proving the truth of statements.

For example, take the statement *"An integer doubled is larger than the integer."*. This statement is false, and we can prove that it is false simply by showing that *2\*(-1) = -2* and that *-2 < -1*. By just using an example we can disprove the truth of the statement.

However, take the statement *"The sum of 2 odd numbers is even."*. This is something that you can see when you run through examples, (3+7 = 10, 21+43=64 etc.) but in order to create this more general statement from these specific examples we must come up with some formal proof. Simply showing these examples is not sufficient. It is very easy to generalise to a statement that is entirely incorrect from small group of examples.

>It is possible to disprove universal statements with examples, but not prove them.

Essentially in maths. We cannot believe a statement just because it seems true, we need to see or write some kind of proof that shows that it is true.

This is also true in computer science. When writing a program we often need to (or at least should) attempt to prove that it obeys some rules that the system requires. This is a quite a large branch of computer science called formal verification.
